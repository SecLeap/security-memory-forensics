# 仓库导航与维护约定

## 使用原则

- 以“可验证、可复现、可追溯”为最低标准沉淀知识。
- 平台、系统版本、工具版本、采集时间和时区必须明确标注。
- 命令先说明对内存、系统稳定性和权限的影响。
- 一条结论应关联证据位置、分析方法和可信度。

## 建议学习顺序

1. `01-基础技术储备`：补齐操作系统与网络底座。
2. `02-终端平台取证`：理解各系统的关键证据位置。
3. `03-内存取证核心`：掌握采集、解析、对象定位和交叉核验。
4. `04-内存取证场景`：练习典型内存现象的分析路径。
5. `05` 至 `06`：形成工具化能力和规范化取证交付能力。

## 12 周学习路线

| 阶段 | 周次 | 学习重点 | 可交付成果 |
| --- | --- | --- | --- |
| 基础 | 1–2 | 虚拟内存、进程、线程、可执行文件与调试符号 | 三个平台的内存对象速查表 |
| 平台 | 3–5 | Windows、Linux、macOS 的内核与用户态内存结构 | 每个平台一份对象关联笔记 |
| 采集 | 6 | 镜像格式、采集副作用、哈希与元数据 | 一套仅用于实验机的采集记录 |
| 解析 | 7–8 | Volatility 3、符号表、基础插件与原始输出保存 | 三份镜像的标准化分析目录 |
| 深入 | 9–10 | 进程隐藏、注入、模块/驱动、socket 与字符串验证 | 两个可复现的异常内存实验 |
| 交付 | 11–12 | 结论链、反证、报告与工具化 | 一份可复核的内存取证报告 |

每完成一周，只将已在隔离实验环境中复现的步骤标为“已验证”；工具输出不能单独构成结论，至少需要两个独立内存视图或证据来源互相印证。

## 推荐学习入口

- [Volatility 3 入门与平台教程](https://volatility3.readthedocs.io/en/latest/)：主分析框架与 Windows/Linux/macOS 示例。
- [Windows 调试与转储文档](https://learn.microsoft.com/windows-hardware/drivers/debugger/)：理解 Windows dump 与符号。
- [Linux 内核文档](https://docs.kernel.org/)：理解内核数据结构、kdump 与调试信息。
- [The Art of Memory Forensics](https://www.wiley.com/en-us/The+Art+of+Memory+Forensics%3A+Detecting+Malware+and+Threats+in+Windows%2C+Linux%2C+and+Mac+Memory-p-9781118825099)：跨平台内存取证经典参考书，结合当前工具文档阅读。
