# 目录约定

## 内容唯一归属

| 内容类型 | 首选目录 | 不应重复放置的位置 |
| --- | --- | --- |
| 操作系统机制与内存对象 | `01-基础技术储备`、`02-终端平台取证` | 工具或场景目录中的长篇原理说明 |
| 镜像采集、符号、解析与结论链 | `03-内存取证核心` | 单个工具手册中的通用流程重复 |
| 异常对象的判读练习 | `04-内存取证场景` | 实验环境的搭建说明 |
| 单一工具的安装、使用、输出与限制 | `05-工具与自动化` | 实验环境目录中的重复安装教程 |
| 虚拟机、网络隔离、FakeDNS/INetSim 服务与联合实验 | `07-恶意软件内存分析实验环境` | 工具目录中的环境拓扑重复 |
| 证据链、报告和归档 | `06-证据管理与报告` | 任意工具手册中的完整报告模板 |
| 官方文档、术语与外部索引 | `10-资料索引` | 多个专题重复维护相同链接 |

## 文件命名

- 专题目录使用两位数字前缀：`01-主题名`；`README.md` 只承担该目录导航和范围说明。
- 单一工具文档使用 `序号-工具名.md`；标题必须写为“工具名 实战手册”。
- 涉及多个工具或平台的操作流程使用“主题名 实战手册”，并链接到单一工具手册，不复制安装细节。
- 新增文档前先搜索现有主题；若内容重叠，优先补充原文档或添加链接。

## 链接规则

- 同级或上级导航使用相对 Markdown 链接；外部软件仅链接官方项目、厂商页面或官方文档。
- 原始输出、镜像、样本和 PCAP 不进入仓库；文档仅引用案例编号、哈希和受控存放位置。
- 工具手册的结论应回链到 `03-内存取证核心` 的标准工作流和 `06-证据管理与报告` 的交付规则。
