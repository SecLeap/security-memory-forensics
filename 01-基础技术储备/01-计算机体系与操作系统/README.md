# 计算机体系与操作系统

## 学习目标

能够解释一个虚拟地址如何落到物理页，并能区分“进程仍在运行”“对象残留在内存”“仅有缓存页”三种不同结论。优先掌握地址空间、分页、页表、进程/线程、加载器、内核对象与交换机制。

## 推荐资料

- [Windows 调试器文档](https://learn.microsoft.com/windows-hardware/drivers/debugger/)：从 dump、符号和内核调试概念切入 Windows。
- [Linux 内核内存管理文档](https://docs.kernel.org/mm/index.html)：理解 Linux 的虚拟内存和页管理。
- [Apple XNU 源码](https://github.com/apple-oss-distributions/xnu)：以源码验证 Mach/XNU 结构；只按目标系统对应版本阅读。

## 最小实验

1. 在隔离虚拟机中启动一个测试进程，记录 PID、线程数、加载模块与虚拟内存区域。
2. 制作镜像后，使用分析框架重新定位这些对象，并记录对象地址和插件输出。
3. 结束测试进程后再次制作镜像，对比可见对象与残留字符串的差异。
