# 标准分析工作流

## 目录与不可变原则

```text
案件或实验编号/
  原始镜像/          # 只读保存，不参与工具写入
  镜像元数据/        # 哈希、采集记录、系统版本、符号来源
  原始输出/          # 每个插件的完整输出
  结构化结果/        # 清洗后的 JSON/CSV，必须可回溯到原始输出
  笔记与报告/        # 假设、验证、反证和结论
```

## 十步流程

1. 计算镜像 SHA-256，登记来源、大小、时间和访问权限。
2. 识别镜像格式、系统、版本、架构和可用符号。
3. 运行信息类插件，保存完整日志与报错。
4. 枚举进程与进程树，标记异常关系但不急于定性。
5. 对标记进程分析线程、命令行、模块、句柄/文件描述符和地址空间。
6. 分析网络对象、内核组件和全局配置对象。
7. 对异常内存区域进行保护属性、映像来源、线程入口和字节上下文验证。
8. 使用第二种枚举路径或另一工具/插件交叉验证。
9. 记录支持结论与反证结论，明确不可见数据和工具限制。
10. 输出结论、方法、原始证据位置与可复现命令。

## 最小命令记录

```text
时间（含时区）｜分析者｜工具及版本｜镜像 SHA-256｜符号来源
命令｜退出码｜完整输出路径｜结果摘要｜复核状态
```
