# 工具实践清单

## 练习环境

准备一台分析工作站和三台隔离虚拟机（Windows、Linux、macOS）。原始镜像只读保存；工具安装、依赖和符号缓存均放在工作站。macOS 虚拟化须遵守 Apple 的许可与硬件要求。

## 实践 1：Volatility 3

1. 从 [Volatility Foundation](https://github.com/volatilityfoundation/volatility3) 获取框架，创建独立 Python 虚拟环境。
2. 运行 `vol -h`，保存版本与可用插件清单。
3. 先对公开 Windows 镜像运行 `windows.info`、`windows.pslist`、`windows.pstree`。
4. 将完整终端输出、镜像 SHA-256、命令和符号缓存状态归档。
5. 再针对 Linux 和 macOS 公开镜像复现对应教程；若符号不匹配，记录失败原因而不是跳过。

## 实践 2：Linux AVML 采集

1. 在授权的 x86_64 Linux 实验机下载 [AVML release](https://github.com/microsoft/avml/releases)，校验发布者提供的哈希/签名。
2. 运行 `avml acquire --help`，确认版本、输出格式与存储空间。
3. 仅在实验机运行 `sudo avml acquire output.lime`；立即计算 SHA-256，并把采集参数写入记录。
4. 在分析工作站用 Volatility 3 先验证 banner/符号，再运行进程和 socket 相关插件。

## 实践 3：Windows dump 边界

1. 在 Windows 实验机阅读 [完整内存转储](https://learn.microsoft.com/windows-hardware/drivers/debugger/complete-memory-dump) 与 [活动内核转储](https://learn.microsoft.com/windows-hardware/drivers/debugger/task-manager-live-dump)。
2. 对比同一测试进程在进程 dump、活动内核 dump、完整 dump 中的可见性；不要把三者的缺失解释成不存在。
3. 使用 [WinDbg](https://learn.microsoft.com/windows-hardware/drivers/debugger/) 验证 dump 基本信息和符号加载，再以 Volatility 做对象分析。

## 完成标准

- [ ] 记录每个工具的来源、版本、哈希、许可证与运行环境。
- [ ] 每个平台至少有一份可重放的命令日志和结构化输出。
- [ ] 每项结论能定位回镜像哈希、命令、插件/工具版本和原始输出文件。
- [ ] 失败的采集、符号、插件也被完整记录。
