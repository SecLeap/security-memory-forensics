# 恶意软件内存分析环境实战手册

本手册将 Win11 分析工具、Ubuntu 模拟服务和 Volatility 3 串成一个可复现的内存分析实验。详细网络拓扑与场景见 [07-恶意软件内存分析实验环境](../07-恶意软件内存分析实验环境/README.md)。

单工具安装与操作链接统一维护在 [Windows11 逐工具手册](06-Windows11-工具安装手册/README.md) 和 [Ubuntu 逐工具手册](07-Ubuntu-工具安装手册/README.md) 中。

## 环境组成

```text
Win11 分析机（静态/动态/内存工具）
  └─ 仅主机/内部虚拟网络 ─→ Ubuntu（FakeDNS / Apache / INetSim / PCAP）
                                      ↓
                              日志、PCAP、服务报告
                                      ↓
                 案例目录 ← Win11 内存镜像、Procmon、进程与网络记录
```

没有 NAT、桥接、共享目录、共享剪贴板、拖放、USB 直通和默认出网路由。只使用无害测试程序、公开教学样本或已获授权材料。

## 每次实验的执行清单

1. 还原 Win11、Ubuntu 至干净快照；检查 IP、DNS、路由、共享功能均符合隔离要求。
2. 选定 Ubuntu 服务模式，确认 DNS/HTTP 等端口没有重复监听，并创建案例日志目录。
3. 在 Win11 记录基线；启动 Procmon/Noriben、Wireshark 和 System Informer/Process Explorer。
4. 运行无害测试程序或授权材料；在关键时刻登记 PID、命令行、模块、线程与网络会话。
5. 导出 Windows PML、PCAP、服务端日志；制作内存镜像并立即记录 SHA-256。
6. 以 Volatility 3 检查镜像信息、进程树、命令行、模块/VAD 和网络对象。
7. 以 PID、五元组、时间和对象地址关联至少两种独立来源；记录支持与反证。
8. 将所有证据复制到分析工作站的案例目录，关闭虚拟机并还原快照。

## 三类最小实验

| 实验 | 工具组合 | 验证重点 |
| --- | --- | --- |
| DNS→HTTP | FakeDNS + Apache + Wireshark + Procmon + Volatility | 域名、五元组、PID、socket 与服务端日志对齐 |
| 多协议模拟 | INetSim + tcpdump + Wireshark + System Informer + Volatility | 请求时间、所属进程、连接是否在镜像中仍可见 |
| 文件/注册表与内存 | Procmon/Noriben + Regshot + DIE/Ghidra + Volatility | 文件/注册表事件、模块、线程与 VAD 的解释链 |

## 交付标准

- [ ] 工具、服务、配置、镜像和输出均有版本/哈希记录。
- [ ] Win11 与 Ubuntu 的时间、时区、IP、端口和快照编号明确。
- [ ] 每个结论能回溯到原始日志、PCAP 或镜像插件输出。
- [ ] 实验结束后环境已还原；未遗留样本、日志或网络连通性。
