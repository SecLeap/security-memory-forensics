# Detect It Easy 安装实战手册

## 获取与安装

从 [DIE 官方 release](https://github.com/horsicq/DIE-engine/releases) 下载适配 Windows 的发布包，解压至 `C:\Lab\Tools\DIE`；记录 release 标签、包哈希和架构。

## 验证与最小使用

仅打开文件副本，记录编译器、保护壳/签名识别、架构和熵等结果。将识别结果写为“工具提示”，不能直接视为定论。

## 内存取证联动

若镜像中出现异常映像或匿名可执行区，DIE 结果可辅助解释其文件格式；仍须用 VAD、线程入口和原始字节验证。

## 使用方法

1. 用 DIE 打开样本副本，记录文件类型、架构、编译器、壳/保护提示和熵。
2. 对提示结果使用另一工具或原始字节复核，不据此执行或修改文件。
3. 保存检测结果、输入哈希和 DIE 版本。

## 实战场景与完成标准

场景：解释为何样本的静态节区与内存中映像布局可能不同。完成标准是报告区分“工具识别提示”和“已通过内存上下文验证的事实”。
