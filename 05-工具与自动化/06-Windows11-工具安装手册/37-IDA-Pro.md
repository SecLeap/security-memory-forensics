# IDA Pro 实战手册

> 适用范围：在隔离分析机中对已获授权的二进制**工作副本**进行静态反汇编、交叉引用与反编译注释，并将静态 RVA 与内存镜像中的运行时 VA 对齐。IDA Pro 不执行样本，亦不替代 Volatility 3/MemProcFS 的内存对象解析。

[IDA Pro](https://docs.hex-rays.com/) 为商业软件。仅通过 My Hex-Rays 下载中心和有效许可取得安装包与 `ida.hexlic`；不得使用破解版本、共享许可文件或来源不明插件。许可证文件、账号信息和授权凭据不进入案例目录、截图或仓库。

## 获取与安装

### 1. 许可与安装校验

Hex-Rays 官方 Windows 安装要求激活命名/计算机许可并下载 `ida.hexlic`，随后通过安装程序部署；许可文件可放在 IDA 安装目录或 `%APPDATA%\Hex-Rays\IDA Pro`。在隔离分析 VM 完成授权与安装，记录但不公开许可证标识。

```powershell
Get-FileHash 'C:\Lab\Installers\IDA\<IDA安装包>.exe' -Algorithm SHA256
Get-AuthenticodeSignature 'C:\Program Files\IDA Professional\ida64.exe' |
  Format-List Status,StatusMessage,SignerCertificate
```

记录 IDA 产品/版本、安装包 SHA-256、许可类型（不含序列号）、安装路径、IDAPython 版本、插件清单和 VM 快照。仅安装官方或经组织审核的插件；案例期间禁用自动更新、在线服务及未知脚本。首次以良性 PE 副本打开并创建本地项目，确认反汇编与反编译视图可用。

## 案例项目与证据边界

```text
C:\Lab\Cases\LAB-001\
├─ 00-原始文件\
├─ 01-工作副本\
├─ 02-IDA-项目\             # .i64/.idb 与派生注释，访问受控
├─ 03-静态分析笔记\
└─ 04-内存镜像与输出\
```

1. 原件只读保管；导入 IDA 的只能是工作副本。
2. 项目名包含案例号与样本短哈希；记录原件/工作副本 SHA-256、导入时间、选择的处理器/位数/文件格式、映像基址和自动分析选项。
3. `.i64/.idb`、导出伪代码、函数列表和截图均为分析派生物；对关键派生文件计算 SHA-256，但不把它们当作原始证据。
4. 不启用会把样本特征、函数元数据或项目内容发送到外部服务的功能，除非已得到明确授权并在记录中说明数据范围。

## 使用方法

### 1. 导入与自动分析

1. 通过 `File > Open` 导入工作副本，确认文件路径与 SHA-256。
2. 检查 IDA 自动识别的文件格式、处理器、位数、字节序、编译器线索和 image base；识别不确定时停止并与 DiE/PEStudio/HxD 复核，而非随意选择架构。
3. 使用默认/最小化自动分析启动，记录分析配置、完成时间和告警。加壳、截断、手工映射或裸内存导出物可能无法形成完整函数图。
4. 保存本地项目，导出函数/字符串或截图前先记录项目文件路径和版本。

### 2. 静态审阅顺序

| 顺序 | IDA 视图/对象 | 记录目标 |
| --- | --- | --- |
| 1 | Imports、Exports、Segments | 导入/导出、节/段范围、image base、可执行区域 |
| 2 | Strings 与 Xrefs | 字符串编码、静态地址、引用函数与原始字节复核位置 |
| 3 | Functions、Graph、Call graph | 函数起点、调用关系和待验证的控制流假设 |
| 4 | Decompiler | 伪代码理解、变量/函数重命名和解释性注释 |
| 5 | Hex View | 与 HxD/PEStudio 一致的原始字节、文件偏移与结构边界 |

注释应区分“工具/分析推断”和“已由内存/动态证据确认”。例如：`调用网络相关导入，待 PCAP 与 socket 验证`，而非直接写为“已联网”。

### 3. 地址换算与内存互证

静态结果以 IDA VA 表示，运行时地址受 ASLR 与映像基址影响。对 PE 模块使用：

```text
RVA = IDA VA - 静态 image base
运行时 VA = 内存中模块基址 + RVA
```

记录三者及来源：IDA image base、Volatility/MemProcFS/x64dbg 的运行时模块基址、目标 RVA。仅当模块版本、SHA-256、PID 和时间关系一致时才比较地址；导出物截断、重定位、不同版本和手工映射会导致不一致。

## 与内存取证的联合分析

1. Volatility 3/MemProcFS 从完整镜像记录 PID、模块、VAD、线程起始地址和文件对象来源。
2. 在 IDA 中打开相同文件版本的工作副本，定位线程起点或模块内 RVA，对照函数、字符串、导入和附近字节。
3. 用 HxD/PEStudio/DiE 验证 PE 结构和文件偏移，用 FLOSS/capa 辅助形成字符串/能力假设。
4. 将静态函数地址、运行时 VA、模块基址、线程 ID、VAD 范围与镜像 SHA-256 汇入案例时间线。

## 常见问题与排错

| 现象 | 处理方式 |
| --- | --- |
| 无法激活/许可错误 | 核对官方许可证位置和授权状态；不使用破解文件或共享凭据 |
| 架构/格式识别错误 | 检查 DiE、PEStudio、HxD 头部和导出边界；记录不确定性 |
| 反编译结果异常 | 反编译是辅助表示；回到汇编、控制流、原始字节和符号/类型信息复核 |
| 地址与内存不一致 | 区分 IDA VA、RVA、运行时 VA，核对 ASLR、版本、采集时间和模块哈希 |
| 插件/脚本报错 | 禁用并记录；仅使用已审核、版本固定的插件/脚本，不从论坛临时下载执行 |

## 实战检查清单

- [ ] IDA 与许可证均来自合法官方渠道，许可证/账号信息未进入项目或仓库。
- [ ] 仅导入工作副本，已记录样本哈希、架构、image base 和分析配置。
- [ ] 静态注释明确区分推断与已验证行为。
- [ ] 关键地址已按 VA/RVA/运行时 VA 规则与 Volatility 3/MemProcFS 对齐。
- [ ] 未启用外部数据共享、未知插件或脚本。
