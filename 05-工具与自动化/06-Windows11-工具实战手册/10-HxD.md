# HxD 实战手册

> 适用范围：在离线分析机中对内存镜像导出物、恢复文件和其他二进制工作副本做**只读字节级复核**。HxD 不是内存采集器和镜像解析器；本手册禁止打开物理磁盘、RAM 编辑器或执行任何写入、替换、擦除、分割/合并操作。

[HxD](https://mh-nexus.de/en/hxd/) 是可处理大文件的十六进制编辑器，提供搜索、比较、校验、导出、书签和统计等功能。官方页面公开的 2.5.0.0 版本较早且列出的系统版本截至 Windows 10；在 Win11 分析机上先以良性文件完成可用性验证。其具备磁盘和内存编辑能力，本仓库仅使用“打开文件副本、查看、搜索、复制偏移、导出报告”的最小权限流程。

## 获取与安装

### 1. 下载与验证

1. 从 [HxD 官方页面](https://mh-nexus.de/en/hxd/) 的下载链接获取 portable 或安装版；优先在隔离分析 VM 使用 portable 版，解压至 `C:\Lab\Tools\HxD\`。
2. 保留下载原件、下载 URL、版本、许可证和 SHA-256；安装版仅部署在分析 VM 并创建快照。
3. 用已知良性文件确认能打开、搜索、复制偏移和关闭而不保存。若 Win11 兼容性异常，记录并改用已批准工具，不要在现场降低安全配置。

```powershell
Get-FileHash 'C:\Lab\Installers\HxD\<发布包>' -Algorithm SHA256
Get-FileHash 'C:\Lab\Tools\HxD\HxD.exe' -Algorithm SHA256
Start-Process -FilePath 'C:\Lab\Tools\HxD\HxD.exe'
```

## 只读证据准备

```text
C:\Lab\Cases\LAB-001\
├─ 00-原始导出物\          # 原始镜像导出物，只读保管
├─ 01-工作副本\            # HxD 打开的副本
└─ 02-HxD-记录\            # 偏移、截图、导出文本、哈希
```

1. 原始导出物保持只读，复制到 `01-工作副本` 后再用 HxD 打开；原件与副本分别计算 SHA-256。
2. 将工作副本设为只读属性，降低误保存风险；HxD 弹出保存提示时选择不保存。
3. 不以 HxD 的编辑、磁盘、RAM、擦除或写入能力处理任何案件对象。

```powershell
$source = 'C:\Lab\Cases\LAB-001\00-原始导出物\candidate.bin'
$copy = 'C:\Lab\Cases\LAB-001\01-工作副本\candidate.bin'
Copy-Item $source $copy
attrib +R $copy
Get-FileHash $source,$copy -Algorithm SHA256 |
  Export-Csv 'C:\Lab\Cases\LAB-001\02-HxD-记录\LAB-001-input-hashes.csv' -NoTypeInformation -Encoding utf8
```

## 使用方法

### 1. 打开与定位

1. 使用 `File > Open` 打开工作副本，确认标题栏路径与案例副本一致。
2. 通过 `Search > Find` 搜索 Hex values、文本字符串或 Unicode 文本；记录搜索模式、编码、是否区分大小写、起始偏移与命中次数。
3. 通过 `Search > Goto` 定位已由 Volatility、MemProcFS、PEStudio、DiE、FLOSS 或 capa 提供的文件偏移/RVA 映射结果。
4. 使用书签标记少量关键偏移；在 `02-HxD-记录` 保存书签名称、偏移、长度、相邻字节和解释，不依赖界面书签作为唯一记录。

常见的**格式识别线索**：PE 文件开头的 `4D 5A`（`MZ`）和后续 `50 45 00 00`（`PE\0\0`）。这只能说明相关字节存在；还需检查偏移关系、PE 头、节表、来源和导出完整性，不能凭两个魔数判断为完整可执行文件。

### 2. 字节、字符串与 PE 结构复核

| 任务 | HxD 操作 | 必须记录 |
| --- | --- | --- |
| 验证导出物完整性 | 检查开头、关键头部、文件尾和已知节表位置 | 文件 SHA-256、偏移、长度、原始字节截图/导出 |
| 复核可疑字符串 | 以 ASCII/Unicode/Hex 搜索，查看前后字节 | 编码、偏移、原始字节、命中总数、所属对象 |
| 对齐静态工具结果 | 用 PEStudio/DiE/capa/FLOSS 给出的地址或字符串定位 | 工具版本、地址类型（文件偏移/RVA/VA）、转换依据 |
| 比较两个副本 | 使用 File compare 或在相同偏移人工复核 | 两个输入哈希、比较范围、差异偏移和长度 |
| 计算辅助校验 | 使用 HxD 校验功能作快速观察 | 以 PowerShell SHA-256 清单作为证据主值 |

RVA、VA 与文件偏移不能混用。对已确认的 PE 节，文件偏移通常按 `PointerToRawData + (RVA - VirtualAddress)` 计算；该关系只在目标 RVA 落入相应节且节表完整时成立。地址不确定时，记录“未确定”并回到 PEStudio/CFF Explorer 或内存映射复核。

### 3. 导出与报告

对需复核的有限范围，使用 HxD 的导出/复制功能生成派生文本或十六进制视图；原始副本不修改。输出文件名应带案例号、输入短哈希、偏移和长度，例如 `LAB-001_a1b2c3d4_off-00001200_len-00000200.txt`。

每项字节证据记录模板：

```text
输入对象：candidate.bin；SHA-256：<hash>
来源：Volatility <插件/对象> 或 MemProcFS <VFS 路径>
HxD：2.5.0.0；只读工作副本
位置：文件偏移 0x00001200；长度 0x200；地址类型：文件偏移
观察：<原始字节/字符串/PE 字段>
复核：<PEStudio/DiE/FLOSS/capa/Volatility VAD/线程>
结论边界：<仅字节存在 / 已由上下文支持>
```

## 与内存取证的联合分析

### 场景一：验证 Volatility 或 MemProcFS 导出物

1. 保存完整 RAM 镜像 SHA-256、导出命令/路径、对象来源、PID、VAD 或文件对象信息。
2. 为导出物创建 HxD 工作副本，记录其哈希和大小。
3. 用 HxD 检查头部、结构边界、字符串和静态工具声称的偏移；用 PEStudio/DiE 判断是否为完整 PE。
4. 回到 Volatility 3 VAD、线程、模块与 MemProcFS 映射视图确认来源。若仅发现片段或头部缺失，报告为“导出物不完整/格式待定”，不补造字节。

### 场景二：解释内存中与磁盘副本的差异

1. 对磁盘工作副本和内存导出物分别计算哈希，选择可比的相同结构范围。
2. 用 HxD 比较字节差异并记录偏移、长度、两个输入哈希和截图/导出。
3. 检查 ASLR、重定位、页保护、映射方式、导出边界和采集时间；差异本身不等于篡改或注入。
4. 仅当 VAD、线程入口、模块关系与原始字节共同支持时，才描述差异的取证意义。

## 常见问题与排错

| 现象 | 处理方式 |
| --- | --- |
| 不慎打开原始文件 | 立即关闭且不保存；重新计算原件哈希，记录操作；后续只使用新工作副本 |
| 找不到字符串/魔数 | 检查编码、导出边界、页缺失、压缩/加壳和地址类型；“未找到”不是无害结论 |
| 偏移与其他工具不一致 | 区分文件偏移、RVA、VA；记录映像基址/节表，排查重定位和导出不完整 |
| 工具提示保存或出现修改高亮 | 取消保存，丢弃该副本，重新复制原件并计算哈希；不要尝试撤销后继续作为证据 |
| 需要查看磁盘/RAM | 不使用 HxD 的磁盘/RAM 编辑器；以镜像/导出物工作副本替代 |

## 实战检查清单

- [ ] HxD 来自官方页面，已记录版本、下载物和可执行文件 SHA-256。
- [ ] 原始对象与工作副本均已计算哈希，工作副本设为只读。
- [ ] 未打开物理磁盘、RAM 编辑器，也未执行任何保存、替换、擦除或写入操作。
- [ ] 每个关键字节证据都有偏移、长度、地址类型、原始字节和来源链。
- [ ] RVA/VA/文件偏移已区分，并与 PE/内存上下文交叉验证。
- [ ] 镜像导出物结论已回填至 Volatility 3/MemProcFS 的 PID、VAD、模块或线程关系。
