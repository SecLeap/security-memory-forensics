# HashMyFiles 实战手册

> 适用范围：隔离 Windows 11 分析环境中，为工具包、样本工作副本、内存镜像、导出物、PML、PCAP 和报告建立可复核的文件级哈希清单。文件哈希证明指定时间所计算字节序列的一致性，不能证明文件来源、运行行为或内存页与磁盘文件完全相同。

[HashMyFiles](https://www.nirsoft.net/utils/hash_my_files.html) 是 NirSoft 的便携式哈希工具，可计算 MD5、SHA-1、CRC32、SHA-256、SHA-384 与 SHA-512，并将结果导出为文本、HTML、XML 或 CSV。Windows 11 案例的默认完整性标识使用 SHA-256；其他算法仅在兼容既有证据或比对需求时附带保留。

## 获取与安装

1. 从 NirSoft 官方页面获取与分析机架构匹配的发布包，原始 ZIP 留存至 C:\Lab\Installers\HashMyFiles\。记录下载 URL、版本、文件名、下载时间、许可证和 SHA-256。
2. 解压至 C:\Lab\Tools\HashMyFiles\。该工具为便携式程序，不应把未知 DLL 放入系统目录，也不应在生产终端安装右键菜单。
3. 用两个自建文本文件和一个已知相同副本验证：工具能显示 SHA-256，导出的清单包含路径、大小、时间和哈希。并用 Windows 内置 Get-FileHash 对至少一个输入抽样复核。

    Get-FileHash 'C:\Lab\Installers\HashMyFiles\<发布包>' -Algorithm SHA256
    Get-ChildItem 'C:\Lab\Tools\HashMyFiles' -Recurse -File | Get-FileHash -Algorithm SHA256
    Get-FileHash 'C:\Lab\Test\known-good.bin' -Algorithm SHA256

安装记录与案例记录要分开：工具自身哈希用于说明分析环境；案例清单用于说明证据对象。升级 HashMyFiles 后保留旧版、旧配置和旧导出物。

## 证据目录与记录字段

    C:\Lab\Cases\LAB-001\
    ├─ 00-原始文件\
    ├─ 01-工作副本\
    ├─ 02-完整性清单\
    │  ├─ LAB-001-原件-SHA256.csv
    │  ├─ LAB-001-镜像-SHA256.csv
    │  ├─ LAB-001-导出物-SHA256.csv
    │  └─ LAB-001-命令与环境.txt
    └─ 04-内存镜像与输出\

每行应至少包含：案例号、对象路径/相对路径、文件名、大小、SHA-256、创建/修改时间（作为文件元数据而非可信行为时间）、计算开始/结束时间、工具版本、操作者和输入来源。将清单自身再计算 SHA-256，并存入同一案例记录。

## 使用方法

### 1. 图形界面：单文件或有限文件集合

1. 启动 HashMyFiles，确认 Options 中已启用 SHA-256；MD5/SHA-1/CRC32 仅在明确的兼容需求下开启。
2. 使用 File 菜单的 Add File 或 Add Folder，或把**工作副本**拖入窗口。不要把整个磁盘、原件目录和输出目录混在一次任务中。
3. 选择视图列：完整路径、大小、SHA-256、创建/修改时间，以及版本支持时的 Hash Start Time、Hash End Time、Hashing Duration。
4. 将结果导出为 CSV 或制表符文本到 02-完整性清单；导出前确认带列标题。文件名需包含案例号、对象集合和算法。
5. 记录输入范围、递归设置、所选算法和导出时间。不要使用删除、回收站、Explorer 右键或在线查询操作。

示例命名：LAB-001-内存镜像-SHA256.csv、LAB-001-pid-1234-导出物-SHA256.csv。哈希计算期间不要移动、覆盖或继续写入被计算对象；对于正在生成的镜像或 PML，必须在采集/捕获停止并文件关闭后才计算。

### 2. 命令行：可重放的清单

HashMyFiles 支持文件、目录、通配符及 CSV/文本/XML 等导出选项。先用单个无害文件验证当前版本的帮助和路径，再对已冻结的案例对象运行；以下示例只计算明确列出的单个镜像并直接输出 CSV。

    Set-Location 'C:\Lab\Tools\HashMyFiles'
    .\HashMyFiles.exe /file 'C:\Lab\Cases\LAB-001\04-内存镜像与输出\memory.raw' /SHA256 1 /MD5 0 /SHA1 0 /CRC32 0 /SHA384 0 /SHA512 0 /scomma 'C:\Lab\Cases\LAB-001\02-完整性清单\LAB-001-镜像-SHA256.csv'

处理有限目录时，先冻结目录内容并在记录中说明递归范围；大规模集合可用该工具的 SaveDirect 模式直接写入导出文件。不要对案例根目录做无边界通配符匹配，因为新生成的报告和日志会混入清单，破坏可复现性。

使用命令行后，保存完整命令、退出状态、工具版本、工具哈希、输入路径和结果文件哈希到 LAB-001-命令与环境.txt。

### 3. 独立复核

对镜像、关键导出物和最终报告，使用另一实现抽样或全部复核。Windows 内置命令示例：

    Get-FileHash 'C:\Lab\Cases\LAB-001\04-内存镜像与输出\memory.raw' -Algorithm SHA256
    Get-FileHash 'C:\Lab\Cases\LAB-001\02-完整性清单\LAB-001-镜像-SHA256.csv' -Algorithm SHA256

若两种工具结果不同，立即停止后续分析，核对是否是同一完整路径、同一时点、同一文件大小和同一算法；保留两个原始输出，不要通过重命名或替换清单“消除”差异。

## 与内存取证的联合使用

| 对象 | 何时计算 | 必须关联的信息 | 不能得出的结论 |
| --- | --- | --- | --- |
| 原始内存镜像 | 采集完成、文件关闭后立即计算 | 采集器、主机、UTC、大小、采集日志 | 不能证明采集器未影响内存 |
| 镜像工作副本 | 复制完成后、分析前 | 原件哈希、复制路径、操作者 | 相同哈希只说明字节相同 |
| Volatility/MemProcFS 导出物 | 导出完成后 | 镜像哈希、插件/命令、PID、VAD/文件对象、地址范围 | 不能证明导出物是完整磁盘文件 |
| PML/PCAP/服务日志 | 捕获停止、文件关闭后 | 抓取接口/过滤器、时区、时间窗 | 不能证明内容未遗漏 |
| 静态工具输出/报告 | 工具运行完成后 | 输入哈希、工具/规则版本、命令 | 不能证明分析判断正确 |

内存场景的推荐顺序：采集完成 → 计算原始镜像 SHA-256 → 复制并计算工作副本 SHA-256 → 使用 Volatility 3/MemProcFS 分析 → 每个导出物立即计算哈希并记录其对象来源 → 对清单和最终报告再计算哈希。不要以进程导出物的哈希取代完整内存镜像哈希。

## 使用边界与风险控制

- 不使用 Open In VirusTotal Web Site、/virustotal 或其他会访问外部服务的功能。即使只提交哈希，也可能泄露样本存在性或案件线索。
- 不启用 Explorer 右键菜单和删除/回收站功能；分析工具应只读处理案例副本。
- 不依赖 MD5、SHA-1 或 CRC32 作为唯一完整性标识；默认记录 SHA-256。
- 不把“相同哈希”写成同一来源、同一行为或同一内存映像；它只支持特定文件字节一致。

## 常见问题与排错

| 现象 | 处理方式 |
| --- | --- |
| 计算时间很长 | 确认对象已冻结，记录 Hash Start/End 时间；不要中断后把不完整清单当结果 |
| 清单缺少文件 | 检查输入范围、递归设置、访问权限和错误项；保存错误信息并重新建立明确输入清单 |
| 同一路径哈希变化 | 停止分析，检查文件是否仍被写入、是否选错副本、大小和时间是否变化；保留两次结果 |
| 不同工具结果不一致 | 比较算法、完整路径、大小和计算时点；不覆盖旧清单，必要时从原件重新复制 |
| 想查询在线情报 | 先按组织数据出境流程取得授权；本手册默认不进行外部查询 |

- [ ] 已在对象完全写入并关闭后计算 SHA-256。

- [ ] 导出清单包含路径、大小、算法、值、时间、工具版本和输入范围。
