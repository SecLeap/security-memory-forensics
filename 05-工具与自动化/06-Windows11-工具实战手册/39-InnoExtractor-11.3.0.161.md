# InnoExtractor 11.3.0.161 实战手册

> 适用范围：隔离的 Windows 11 分析虚拟机中，对已获授权的 Inno Setup 安装包工作副本进行离线结构查看与受控提取。本手册不运行安装程序、不运行提取出的文件，也不将提取物等同于原始证据。

[InnoExtractor](https://www.havysoft.cl/) 是 HavySoft 提供的 Inno Setup 安装包查看/提取工具。截图中的 InnoExtractor_v11.3.0.161.7z 对应历史版本；历史压缩包名称本身不能证明来源或完整性，必须以获取记录、发布页信息、数字签名（如有）和 SHA-256 建立工具证据链。

## 获取与安装

1. 只从 HavySoft 官方页面获取当前或可追溯的发布包。若案例要求使用用户已保存的 v11.3.0.161，原压缩包留在 C:\Lab\Installers\InnoExtractor\；无法核验来源时，标记为“历史工具，来源待复核”。
2. 在隔离 VM 解压至 C:\Lab\Tools\InnoExtractor\；不要覆盖旧版工具目录。记录版本、安装包/主程序哈希、签名状态和 VM 快照。
3. 用自建无害 Inno Setup 测试包验证“可打开并显示文件清单”；不点击安装，也不提取或执行未知文件。

    Get-FileHash 'C:\Lab\Installers\InnoExtractor\InnoExtractor_v11.3.0.161.7z' -Algorithm SHA256
    Get-ChildItem 'C:\Lab\Tools\InnoExtractor' -Recurse -File | Get-FileHash -Algorithm SHA256

不要启用工具中的在线查询、VirusTotal 或自动提交样本类功能；案例材料、内部路径和提取物均不得未经授权离开隔离环境。

## 证据准备与目录

    C:\Lab\Cases\LAB-001\
    ├─ 00-原始文件\                 # 原始安装包，只读
    ├─ 01-工作副本\                 # 仅供 InnoExtractor 打开的副本
    ├─ 02-提取派生物\InnoExtractor\ # 文件、脚本和清单
    ├─ 03-静态识别\                 # PE-bear、DiE、FLOSS 等输出
    └─ 04-内存镜像与输出\           # 镜像、Volatility/MemProcFS 原始输出

开始前为原件和工作副本计算 SHA-256，并记录来源、大小、时间、操作者、InnoExtractor 版本与工具哈希。提取目录必须是新的、案例专用目录；提取物是派生物，应另行计算哈希和保留目录清单。

## 使用方法

### 1. 只读查看与初步分流

1. 启动 InnoExtractor，通过“打开”选择 01-工作副本中的安装包；不要在资源管理器中双击安装包。
2. 记录安装包识别结果、文件数量、目录结构、嵌入脚本/配置是否可见，以及工具报告的解析错误或不支持版本。
3. 标记可执行文件、DLL、驱动、脚本、压缩包、配置、证书和资源；仅以名称/路径形成待验证假设。
4. 保存包含工具版本、输入短哈希、文件列表和时间的截图或导出清单至 03-静态识别\。

推荐记录：输入文件与 SHA-256；工具版本与工具 SHA-256；观察到的容器/文件清单/脚本线索；解析错误；状态“仅结构线索，未执行”。

### 2. 受控提取

1. 新建 02-提取派生物\InnoExtractor\，写入输入哈希、目标目录、操作人和开始时间。
2. 选择“提取到”该目录；优先按清单选择所需对象。需要全量提取时，先记录原因和预计文件数。
3. 保存提取日志与失败项；绝不从提取目录启动 EXE、DLL、脚本、快捷方式或二级安装包。
4. 生成派生物哈希清单。

    $out = 'C:\Lab\Cases\LAB-001\02-提取派生物\InnoExtractor'
    Get-ChildItem $out -Recurse -File |
      Get-FileHash -Algorithm SHA256 |
      Export-Csv "$out\LAB-001-InnoExtractor-派生物-SHA256.csv" -NoTypeInformation -Encoding utf8

提取失败、密码提示、损坏或格式不支持均应原样记录。不得为“拿到文件”而运行安装包、替换文件头、修改脚本或使用在线解包服务。

## 结果复核与内存取证联动

| 观察/派生物 | 合法下一步 | 结论边界 |
| --- | --- | --- |
| 文件清单、目录与脚本线索 | HxD、FileInsight 复核字节；记录路径与哈希 | 只能说明安装包中存在相关对象 |
| 提取出的 PE/DLL | 用 [PE-bear](41-PE-bear-0.7.2.md)、PEStudio、DiE 做静态结构检查 | 不代表该模块已经执行或加载 |
| 提取出的 PyInstaller 文件 | 用 [pyinstxtractor-ng](42-pyinstxtractor-ng.md) 做派生物静态提取 | 每层对象都需重新建立哈希关系 |
| 内存中导出的安装器/文件对象 | 先记录镜像哈希、PID、文件对象/VAD、导出命令和哈希，再比对静态清单 | 文件差异可能来自版本、截断或采集时点 |

InnoExtractor 只能帮助解释“导出的对象是否像 Inno Setup 容器、静态包含什么”。进程是否执行、何时写入/加载，应由 Volatility 3/MemProcFS 的对象来源、Procmon 记录和时间线独立支持。

## 常见问题与排错

| 现象 | 处理方式 |
| --- | --- |
| 无法打开或提示不支持 | 记录完整报错、输入哈希和工具版本；用原始字节确认是否完整 Inno Setup 文件，不从第三方站补组件 |
| 提取结果缺失/报错 | 保留清单与日志；核对原件大小、工作副本哈希和目标目录权限，不能将缺失项假定为不存在 |
| 历史版本无法核验 | 不用于得出关键结论；保留为待核验工具，优先在可追溯版本上复现并记录差异 |
| 提取物被安全软件告警 | 不关闭生产防护或建立永久排除；仅在隔离 VM 重新核验工具/样本来源，并保留告警记录 |

- [ ] 原始安装包保持只读，InnoExtractor 只打开工作副本。
- [ ] 已保留文件清单、提取日志/失败项和目标目录哈希清单。
- [ ] 未运行安装包、提取出的载荷、脚本或二级安装器。
- [ ] 已将静态发现与内存中的 PID、VAD/文件对象、采集时间明确分开。
