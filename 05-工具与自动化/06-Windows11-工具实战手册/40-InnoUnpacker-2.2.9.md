# InnoUnpacker 2.2.9 实战手册

> 适用范围：隔离 Windows 11 分析虚拟机中，对已授权 Inno Setup 安装包工作副本做离线查看和受控提取。InnoUnpacker/innounp 的输出是派生物；本手册不运行安装包、提取物或任何脚本。

截图中的 InnoUnpacker_v2.2.9.zip 未附维护者、原始发布地址或校验信息。压缩包名称与版本号不足以确认作者和供应链安全，因此本手册把它视为“**来源待核验的历史工具**”：未取得可信发布页或维护者校验值前，不得把它作为唯一工具或用于关键结论。

## 获取与安装

1. 将用户提供的原始 ZIP 只读留存到 C:\Lab\Installers\InnoUnpacker\，记录取得方式、时间、文件名和 SHA-256。
2. 取得可信来源后，比较发布页面的版本、签名或校验值；如无法比较，记录“来源待核验”，并优先用 [InnoExtractor](39-InnoExtractor-11.3.0.161.md) 在可追溯环境中复核结果。
3. 仅在隔离 VM 解压至 C:\Lab\Tools\InnoUnpacker-2.2.9\；不要替换系统目录、不要把未知目录加入全局 PATH。
4. 用自建无害 Inno Setup 测试包验证界面是否能读取文件清单。不要用未知安装包验证安装，也不要通过运行提取内容确认“成功”。

    Get-FileHash 'C:\Lab\Installers\InnoUnpacker\InnoUnpacker_v2.2.9.zip' -Algorithm SHA256
    Get-ChildItem 'C:\Lab\Tools\InnoUnpacker-2.2.9' -Recurse -File | Get-FileHash -Algorithm SHA256

若该版本实际是某个 innounp 命令行工具的 GUI 前端，须分别记录 GUI 与底层 innounp 程序的版本和哈希。不要根据网络文章猜测参数；确需命令行时，先保存当前程序 --help 或帮助页的原始输出。

## 证据准备

    C:\Lab\Cases\LAB-001\
    ├─ 00-原始文件\                   # 原始安装包，只读
    ├─ 01-工作副本\                   # InnoUnpacker 输入
    ├─ 02-提取派生物\InnoUnpacker\    # 受控提取目录
    └─ 03-静态识别\                   # 清单、截图、后续静态工具输出

输入记录至少包括：案例号、原始/工作副本 SHA-256、来源、工具 ZIP/主程序 SHA-256、工具/底层组件版本、操作时间和目标目录。将提取前后的目录树和派生物 SHA-256 一并保存。

## 使用方法

### 1. 图形界面查看

1. 以普通用户权限启动工具；通过“打开”或拖放选择 01-工作副本中的安装包。禁止双击安装包本身。
2. 在未提取前记录可见的安装包信息、文件路径、大小、属性与解析/兼容性提示。
3. 若界面提供“运行”“测试安装”“打开文件”或在线查询，均不要使用。
4. 截图/导出文件清单时，包含输入短哈希、工具版本与 UTC 时间；保存到 03-静态识别\。

### 2. 受控提取与派生物留档

1. 创建新的案例目录 02-提取派生物\InnoUnpacker\；提取前记录目标目录为空及输入哈希。
2. 使用界面将选定项目或全部项目提取至该目录。全量提取必须写明目的；不向原件目录、工具目录或桌面提取。
3. 保存工具的完成/错误信息、已选项和文件列表。提取操作绝不包含打开、运行或预览提取物。
4. 对所有提取物生成哈希清单，并将每个对象与安装包输入哈希关联。

    $out = 'C:\Lab\Cases\LAB-001\02-提取派生物\InnoUnpacker'
    Get-ChildItem $out -Recurse -File | ForEach-Object {
      $h = Get-FileHash $_.FullName -Algorithm SHA256
      [pscustomobject]@{ Path=$_.FullName; Length=$_.Length; SHA256=$h.Hash }
    } | Export-Csv "$out\LAB-001-InnoUnpacker-派生物-SHA256.csv" -NoTypeInformation -Encoding utf8

### 3. 命令行前端（仅在本机帮助已核验时）

不同打包版本、GUI 前端与底层组件的选项可能不同。本仓库不固定猜测的提取参数。仅按以下步骤留档：

1. 对无害测试包运行该版本的帮助命令，保存标准输出、可执行文件哈希和返回码。
2. 确认“列表/查看”和“指定输出目录”的实际语法后，先对工作副本生成清单，再单次提取到案例派生物目录。
3. 保存完整命令、终端输出、输入/输出哈希和目录树；禁止使用任何会执行安装器或提取物的选项。

当帮助无法确认语义或程序来源无法验证时，停止使用命令行路径，转用可追溯的 InnoExtractor 并在报告中说明原因。

## 与内存取证和其他工具联动

| 场景 | 操作顺序 | 证据边界 |
| --- | --- | --- |
| 从内存文件对象导出的疑似安装器 | 记录镜像哈希、PID/文件对象、导出命令与哈希 → 静态确认容器 → InnoUnpacker 工作副本查看 | 仅支持“导出物可被解析为/含有某对象” |
| 安装包中的 PE 载荷 | 受控提取 → PE-bear/DiE/PEStudio 静态检查 → 将哈希与进程模块/VAD 对齐 | 静态存在不等于载荷曾被执行 |
| 嵌入脚本/配置 | 保留原始提取字节、编码和路径 → HxD/FileInsight 离线阅读 | 不执行脚本，不把文本直接当运行时命令 |
| 两种提取工具结果不同 | 固定同一工作副本与哈希 → 保存两份清单 → 比较文件数、路径和哈希 | 差异可能来自兼容性、版本或解析策略 |

## 常见问题与排错

| 现象 | 处理方式 |
| --- | --- |
| 工具来源或版本无法确认 | 标记“来源待核验”，不作为唯一依据；改用可追溯工具复核 |
| 无法识别安装包 | 保存报错与输入哈希，检查文件头、大小和是否截断；不运行安装包测试 |
| 未知命令行参数 | 只保存本机帮助输出；不要复制不明网页命令或试错执行 |
| 提取目录出现可执行文件 | 计算哈希、静态分流；绝不双击或加载 |
