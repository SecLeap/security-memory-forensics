# 证据管理与报告

## 内存取证报告骨架

1. **范围与授权**：镜像编号、系统、时间范围、问题与不可覆盖内容。
2. **材料与完整性**：镜像路径、大小、SHA-256、采集工具、符号/调试信息来源。
3. **方法**：分析工具版本、插件、参数、输出位置与交叉验证方法。
4. **发现**：对象地址、所属进程/内核组件、观察到的字段、正常对照。
5. **结论与可信度**：支持证据、反证、替代解释、已知工具/镜像局限。
6. **附件**：命令日志、完整原始输出、对象关联表和哈希清单。

## 结论措辞

使用“镜像中观察到”“由两个独立枚举路径验证”“因符号不匹配无法确认”等可复核表述；避免把单个字符串、单个规则命中或空插件输出写成确定性事实。
